# Space Complexity

We will learn how to **calculate** the space complexity of a piece of code using Big O notation, the same way we did for time complexity.

## Concept
Space complexity in Big O notation measures how much extra memory an algorithm needs as its input grows.

```
Space Complexity = Auxiliary Space + Space used for input values
```

**Auxiliary space** is any extra or temporary memory the algorithm allocates on top of its input (new arrays, extra variables, a recursive call stack). **Input space** is the memory the input itself already occupies.

>[!NOTE]
> In practice, when people talk about an algorithm's space complexity they usually mean its **auxiliary space complexity**: the *extra* space the algorithm adds, since the input has to exist in memory regardless of which algorithm runs on it.

### Memory used by statements
Just like every line of code has a time cost, every declared variable has a memory cost. We can estimate the space for each statement by first identifying what it creates.

1. A single fixed-size variable (int, boolean, a reference)

```java
int x = 1;      // 1 int slot
boolean flag;   // 1 boolean slot
```
The space complexity for the above code is **O(1)**

2. An array or list of size `n`

```java
int[] arr = new int[n];   // n int slots
```
The space complexity for the above code is **O(n)**

3. A 2D array of size `n x n`

```java
int[][] grid = new int[n][n];   // n * n int slots
```
The space complexity for the above code is **O(n²)**

### Big O rules for space

Big O follows the rules below when calculating space complexity.

1. **Ignore exact byte counts**

```java
public static int calculateSum(int x, int y) {
    int a = x + y;      // a: 1 int
    return a;
}
```
- `x`, `y`, and `a` are each a single, fixed-size value, none of them grow with any input size.
- No matter how large `x` and `y` are as *numbers*, the amount of memory used stays the same (a few fixed-size int slots).

The space complexity of `calculateSum` is **O(1)**.

>[!NOTE]
> Big O deliberately ignores exact byte counts (like "12 bytes"). What matters is *how memory usage scales* as input grows, not the literal number of bytes at one input size.

2. **Only count auxiliary space, not input space**

```java
public int sum(int[] array) {
    if (array == null || array.length == 0) {
        return 0;
    }
    return Arrays.stream(array).sum();
}
```
- The `if` check uses a constant amount of space: **O(1)**.
- `Arrays.stream(array)` does not copy the array; it creates a lazy view over it, so it adds only **O(1)** extra space.
- `.sum()` accumulates into a single value: **O(1)**.

The **auxiliary** space complexity of `sum` is **O(1)**, even though the input array itself is O(n). The input space is not counted as "extra" because it exists whether or not this function runs.

3. **Space grows with the data structures created, not the number of variables**

```java
public int[] createArray(int n) {
    int[] arr = new int[n];        // O(n)
    for (int i = 0; i < n; i++) {  // O(1)
        arr[i] = i;
    }
    return arr;
}
```
- `n`: a single fixed-size value, O(1).
- `arr`: an array of size `n`, so it grows linearly, **O(n)**.
- `i`: a single fixed-size loop counter, O(1).

The space complexity of `createArray` is **O(n)**. A single extra array dominates any number of O(1) helper variables sitting alongside it.

## Example
1. **Bubble Sort**

```java
public static void bubbleSort(int[] arr) {
    int n = arr.length;                      // O(1)
    for (int i = 0; i < n - 1; i++) {         // O(1)
        for (int j = 0; j < n - i - 1; j++) { // O(1)
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];             // O(1)
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```
The sort is performed **in place**, no new data structures grow with `n`, so its auxiliary space complexity is **O(1)**.

2. **Insertion Sort**
```java
public static void insertionSort(int[] arr) {
    int n = arr.length;              // O(1)
    for (int i = 1; i < n; i++) {    // O(1)
        int key = arr[i];             // O(1)
        int j = i - 1;                // O(1)
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```
Like bubble sort, insertion sort shifts elements directly inside the input array using only a handful of fixed-size helper variables (`key`, `j`), so it is also **O(1)** auxiliary space.
